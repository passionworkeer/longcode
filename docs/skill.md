# longcode Skill - 长时间自主开发代理

> 创建日期: 2026-03-03
> 版本: 1.0.0

---

## 🎯 概述

`longcode` 是一个长时间自主运行的开发技能，能够：
1. **深度理解需求** - 交互式收集所有细节
2. **精细规划** - 将大任务拆解为微小、可执行的任务单元
3. **自主执行** - 启动子代理团队并行开发
4. **自动验收** - 主代理审核每个任务
5. **持续推进** - 完成一个任务后自动进入下一个，不中断

## 🔧 技能配置

### 技能元数据
```yaml
name: longcode
version: 1.0.0
description: 长时间自主开发代理 - 从需求到部署的自动化
author: Claude + User
category: automation
```

### 参数
| 参数 | 类型 | 描述 | 默认值 |
|------|------|------|--------|
| max_task_duration | number | 单任务最大时长(分钟) | 120 |
| enable_git | boolean | 是否自动提交 | true |
| parallel_agents | number | 并行代理数量 | 3-6 |
| review_required | boolean | 是否需要代码审查 | true |
| security_scan | boolean | 是否进行安全扫描 | true |

### 中断恢复机制
```swift
// 如果会话中断，恢复执行
longcode --resume

// 查看当前进度
longcode --status

// 暂停执行
longcode --pause

// 取消执行
longcode --cancel
```

## 🛡️ 安全与质量控制

### 自动检查点
每个任务完成后自动执行：
1. ✅ 编译检查 - 确保代码能构建
2. ✅ 测试通过 - 单元测试必须通过
3. ✅ 安全审查 - 无高危漏洞
4. ✅ 代码审查 - 符合规范
5. ✅ Git 提交 - 有意义的提交信息

### 回滚机制
```swift
// 如果任务失败
1. 记录失败原因到执行日志
2. Git 回滚该任务的更改
3. 标记任务为 failed
4. 继续下一个不依赖的任务
5. 最后生成失败报告
```

## 🚀 实现要点

### 技术实现
1. **状态持久化** - 将执行状态保存到 `.longcode_state.json`
2. **代理编排** - 使用 Agent tool 启动子代理
3. **进度追踪** - 实时更新计划文档
4. **错误恢复** - 支持中断后恢复
5. **Git 集成** - 每个任务自动提交

### 关键设计决策
| 决策点 | 选择 | 原因 |
|--------|------|------|
| 任务粒度 | < 2小时 | 快速反馈，易于并行 |
| 并行策略 | 依赖驱动的任务组 | 最大化效率 |
| Git 提交 | 每任务一提交 | 便于回滚 |
| 用户交互 | 执行期间零交互 | 避免中断 |
